网站技术说明与变迁史

Part.1 网站部署

  • 主线路： www.iuoyt.com

    此线路作为主线路，解析至位于华北地区的国内服务器，速度较快，较为稳定，境内平均响应时间：36.7ms。境外请求则解析至Vercel线路。此线路质量：联通>移动>电信。
    由于一些特殊原因，暂时无法访问到此线路，临时转由Cloudflare代理该线。

  • 备用线路1： vercel.iuoyt.com

    Vercel节点广泛分布于全球各地，在国内也有比较快的访问速度，平均响应时间：93.6ms。对于境外请求，将通过提供Cloudflare解析，平均响应时间：21.4ms。此线路质量：移动>联通>电信。由于一些特殊原因，境内可能出现无法访问的情况。

  • 备用线路2： Gitee（暂不提供）

    部署在Gitee Pages的分站，环境是位于华北地区的百度云，境内平均响应时间30ms。
    由于部署环境原因，暂不提供此线路访问。

  • 备用线路3： github.iuoyt.com

    部署在Github Pages的分站，境内平均响应时间126.3ms。此线路质量：联通>电信>移动。

  • 备用域名4： zeabur.iuoyt.com

    部署在Zeabur的分站，境内平均响应时间106.2ms，境外平均响应时间134.4ms。此线路质量：联通>电信>移动。

  • 备用线路5： cloudflare.iuoyt.com / cf.iuoyt.com

    部署在Cloudflare Pages的分站，境内平均响应时间：216ms，境外平均响应时间：21.4ms。此线路质量：移动>电信>联通。

Part.2 博客入口

因为博客分站（线路）较多，为了方便各地区用户的访问，提供入口网站，可以自动选择最优线路访问博客。

  • 入口1： blog.iuoyt.com

  • 入口2： test.iuoyt.com / verceltest.iuoyt.com

    由于一些特殊原因，境内可能出现无法访问的情况。

  • 入口3： githubtest.iuoyt.com

Part.3 资源托管

  • 平台1： 阿里云OSS

    网站资源Bucket位于华南地区，在国内获取资源有着极快的下载速度，可靠性高，基本不会出现资源获取困难的问题。

  • 平台2： Microsoft Onedrive （自建CDN）

    用于存放部分低频获取的大型资源，访问速度一般，但较为稳定。

  • 平台3： Github

    极其不稳定，但是Free😋。

Part.4 网站支持

  • 博客框架： Hexo
  • 博客主题： Butterfly
  • 开发语言： HTML + CSS + JavaScript + Pug + YML + Markdown
  • 评论系统： Twikoo + Vercel + MongoDB
  • CDN加速： 阿里云CDN + 阿里云DCDN
  • 定位服务： 腾讯位置服务
  • 天气服务： 和风天气
  • 地图服务： 高德开放平台
  • 统计服务： 百度统计 + 51.la
  • 更新日志： 网站更新日志

Part.6 网站变迁史

小站变迁史

2023-07-14

  • 大二结束的暑假，终于着手搭建自己的个人站点，其实本来的想法是只做个起始页，但想了一两年一直没实践，这次偶然在B站看到了Hexo博客搭建教学，于是就先搭了个博客。参考了大佬的教学，把博客框架搭起来了，23年7月12日，网站正式开始运行了。V0.1，启动！
  • 捣鼓了两天，发现了个致命问题，需要整体重构😭。遂放弃先前搭建的内容，顺便用了FomalHaut大佬的开源项目😋，感谢大佬的开源，直接完成了大部分美化工作！

2023-07-20

  • 陆陆续续修复了许多bug，初步完成了魔改和美化，进入V0.2，邀请了某H姓男子开始了站点的一测。
  • 提交了域名备案申请，把博客部署到了服务器上，做了个境内的分站。
  • 把网站的大部分静态资源转移到了自建cdn上，进入网站终于不需要再加载一分钟了。

2023-08-02

  • 部署了Github Pages分站。
  • 想把Butterfly版本更新下，然后就成功的出现了大问题，我还没备份😭。
  • 完成了SITV内容的接入，终于完成了重构旧站点的目标。
  • 第一次域名备案没通过，做了些修改，再次提交了备案。今天终于通过啦。

2023-08-07

  • 完成了公安备案。
  • 完成了页面路径的重构，现在地址栏路径看起来正常多啦。再修了好多bug，然后网站框架就算是成型啦。之后应该不会做太大的修改了。进入二测！（萌备号就是二测日期嘿嘿）
